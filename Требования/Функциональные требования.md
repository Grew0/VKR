# Функциональные требования к системе
## Общие положения
Программное средство предназначено для анализа дампов памяти ArkVM TS Sta и должно обеспечивать выполнение следующих функций:
### Требования к загрузке и парсингу данных
1) Система должна поддерживать загрузку дампов кучи, сгенерированных виртуальной машиной ArkVM Sta TS в ходе выполнения программы;
2) Система должна брать файл дампа кучи из файловой системы по пути, указанному пользователем;
3) Система должна осуществлять обработку содержания файла для дальнейшей визуализации;
### Требования к визуализации
1) Система должна отображать следующие данные, полученные из дампа:
	1) Заполненность регионов
	2) Тип регионов
	3) Дерево объектов с возможностью перемещения по нему
	4) Плотность ремсетов для регионов
    5) Битсеты живых и помарканных объектов для регионов
	6) Классы, которые содержатся в ВМ и их структура

### Требования к статистике объектов
1) Система должна отображать топ самых "тяжелых" объектов
2) Система должна отображать топ классов по количеству объектов в них
3) Система должно отображать топ классов по суммарному объему занимаемой памяти
